C. KRİTİK EKSİKLER (öncelik 1 — hem design hem frontend gerekir)
================================================================================

1. TAKİP LİSTESİ (WATCHLIST) ★ En belirgin boşluk
   Backend: GET/POST/DELETE /api/watchlist, GET /api/watchlist/signals
   Design:  Yok (sidebar'da nav yok, sayfa prototipi yok)
   Frontend: api.ts'te metod yok, rota yok, UI yok

   Önerilen tasarım (Investing.com / TradingView benchmark):
   • Sidebar'a "Takip Listesi" ikonu (watch / star)
   • Tablo veya kompakt kart grid: sembol, logo, fiyat, % değişim, sinyal badge
   • Portföy dışı sembol ekleme (StockSymbolSearch)
   • Sürükle-bırak sıralama, çoklu liste (ileride)
   • Hızlı aksiyonlar: detaya git, alarm kur, portföye ekle
   • /api/watchlist/signals ile hafif teknik skor kolonu (LLM'siz)

2. DERİN ANALİZ SAYFASI / PANELİ (Deep Analysis)
   Backend: POST /api/analysis/deep/{symbol}, GET .../cached (5 ajan + LLM sentez)
   Design:  Yok
   Frontend: Yok (analyzeSymbol API tanımlı ama hiçbir sayfa çağırmıyor)

   Önerilen tasarım:
   • StockDetail'de "Derin Analiz" sekmesi veya tam sayfa
   • Ajan kartları: Temel, Makro, Teknik, Haber, Risk — skor + özet
   • Sentez bölümü (markdown), son güncelleme, yenile butonu
   • Telegram'a gönder toggle (backend ?telegram=true)
   • Yükleniyor / önbellek göstergesi

3. KEŞİF MOTORU TASARIMI
   Backend: POST /api/discovery/scan, GET /api/discovery/latest
   Frontend: DiscoveryPage tam çalışıyor
   Design:  Prototip yok; shell.jsx sidebar'da Keşif yok

   Yapılacak: discovery.jsx prototipi + shell nav güncellemesi + tasarım sistemi dokümanı

4. HİSSE LİSTESİ SAYFASI (Stocks / Portfolio Table View)
   Design shell: "Hisseler" → ilk hisse detayına gider (ayrı liste yok)
   Frontend: Sidebar "Hisseler" → /stock/{ilkSembol}; bağımsız liste sayfası yok

   Önerilen (Yahoo Finance / Finviz):
   • Portföy tablo görünümü: ticker, maliyet, güncel, P&L, ağırlık %, sinyal
   • Sıralama/filtre (getiri, sektör, sinyal)
   • Tablo ↔ kart görünümü toggle
   • Dashboard kartları buraya taşınabilir veya dashboard sadeleşir

5. KAYIT (REGISTER) EKRANI TASARIMI
   Frontend: RegisterPage var
   Design: landing.jsx yalnızca login; register akışı tasarlanmamış

6. DESIGN ↔ FRONTEND NAV UYUMSUZLUKLUKLARI
